# 脑洞基建文·知识库 (Knowledge Base)

写作时随时查阅的弹药库。共 6 个模块。

## 模块 A：硬核基建逻辑 (Realism Anchor)

哪怕写魔法世界，以下逻辑也不能丢，否则会飘。

### A1. 物资流转全链条

| 需求 | 完整产业链 | 常见偷懒写法（避免） |
|------|-----------|-------------------|
| **吃** | 选种→开荒→灌溉→施肥→收割→脱粒→储存→防腐→加工→烹饪 | "主角种了一片田，大家不饿了" |
| **穿** | 原料获取（棉/麻/兽皮/蚕丝）→纺纱→织布→裁剪→缝制→染色 | "主角织了布，大家有衣服了" |
| **住** | 选址→地基→建材制备（伐木/烧砖/和泥）→搭框架→砌墙→封顶→内装 | "主角建了座房子" |
| **行** | 路线勘测→清障→路基夯实→铺路面→排水沟→路标→载具制造 | "主角修了条路" |
| **用水** | 找水源→检测水质→挖渠/打井→过滤→消毒→储存→分配管网 | "主角找到了水" |
| **用电/能源** | 确定能源（煤/水力/风力/魔力）→发电装置→传输线路→变压/稳压→终端设备 | "主角发了个电" |

**核心原则**：产业链。为了造汽车，先得建炼钢厂；为了炼钢，先得挖煤矿和铁矿；为了挖矿，先得修通往矿区的路。每省略一环，可信度下降一层。

### A2. 社会组织逻辑

当人口超过 50 人时，必须考虑以下问题：

| 维度 | 关键问题 | 如果不解决会怎样 |
|------|---------|----------------|
| **分工** | 谁种地/打铁/记账/巡逻/教书？ | 所有人什么都干，效率极低 |
| **分配** | 平均分配 or 按劳分配 or 按需分配？ | 要么饿死懒人，要么养出懒人 |
| **法律** | 偷窃/伤人/叛逃怎么判？ | 丛林法则，强者欺弱 |
| **货币** | 用什么当一般等价物？ | 以物易物效率极低，经济发展卡死 |
| **信仰** | 原住民信什么？主角怎么处理？ | 宗教冲突，或被当成异端 |
| **阶级** | 新来的人和老人权利一样吗？ | 内部矛盾激化 |
| **继承** | 主角不在的时候谁管事？ | 一旦主角出事，全盘崩溃 |

### A3. 科技树层级与前置条件

| Tier | 代表技术 | 文明阶段 | 前置条件 |
|------|---------|---------|---------|
| 0 | 石器/木器/火 | 原始 | 无 |
| 1 | 陶器/青铜器/初级农业 | 部落 | 稳定火源+矿石发现 |
| 2 | 铁器/水泥/纺织机/水车 | 封建/手工业 | 高温冶炼+水利工程 |
| 3 | 蒸汽机/火药/印刷术/玻璃 | 工业早期 | 精密金属加工+化学基础 |
| 4 | 电力/内燃机/化工/无线电 | 工业/现代 | 发电能力+绝缘材料+石油提炼 |
| 5 | 核能/计算机/基因工程 | 信息时代 | 理论物理+半导体+精密仪器 |

**注意**：
- 主角可以"知道怎么做"，但不代表"做得出来"——缺材料/缺工具/缺人手都是合理的卡点
- 科技跳级最多跨 2 个 Tier，且必须用金手指或特殊资源来解释
- 跨 Tier 后必须有"填坑"章节——补全跳过的基础设施

### A4. 常见建设项目·技术细节速查

| 项目 | 核心原理 | 最简实现方案 | 常见坑 |
|------|---------|-------------|--------|
| **水泥** | 石灰石+粘土高温烧制→磨粉→加水凝固 | 找石灰岩烧成生石灰，混粘土 | 需要1000°C+高温，一般篝火不够 |
| **玻璃** | 石英砂+纯碱高温熔化 | 河沙+草木灰烧制 | 极难做到透明，最初只能做半透明的 |
| **肥皂** | 油脂+碱（草木灰水）皂化反应 | 动物油脂煮碱水 | 碱的浓度不好控制 |
| **火药** | 硝石+硫磺+木炭 | 找天然硝石（厕所墙、洞穴壁） | 配比不对会不燃/炸膛 |
| **纸张** | 植物纤维打浆→抄纸→干燥 | 树皮/竹子泡烂捣碎 | 工序繁琐，干燥需要时间 |
| **酒精** | 淀粉发酵→蒸馏 | 粮食/水果发酵 | 蒸馏需要密封容器和温度控制 |
| **抗生素** | 青霉素来自青霉菌 | 发霉面包提取（极简版） | 纯度极低，提纯需要化学知识 |

## 模块 B：脑洞生成器 (Brainhole Generator)

### B1. 反常识物理规则种子库

从以下清单中挑选或组合，生成世界设定：

**空间类**
| 规则 | 对基建的影响 |
|------|-------------|
| 重力是液体，会流动 | 建筑必须"浮"在重力上，或锚定在底层 |
| 空间会周期性折叠 | 两个相距100里的地方每天有1小时直接相邻 |
| 影子是实体 | 白天建筑会产生可触摸的"影子建筑" |
| 地面以下3米全是虚空 | 所有建筑只能建在地表，不能挖地基 |
| 建筑物会缓慢移动 | 每天早上你的房子可能挪了10米 |

**时间类**
| 规则 | 对基建的影响 |
|------|-------------|
| 时间流速因海拔而异 | 山顶1天=山脚1年，可用于加速发酵/生长 |
| 每周二时间倒流1小时 | 建好的东西会被"拆掉"1小时的量 |
| 死物不受时间影响 | 食物永远不会腐烂，但也不能发酵/酿酒 |
| 夜晚时间是白天的10倍 | 夜间是主要生产时间 |

**物质类**
| 规则 | 对基建的影响 |
|------|-------------|
| 声音会凝固成实体 | 锤击的"叮"可以收集成钉子 |
| 颜色有温度 | 红色的东西很烫，蓝色的东西很冷 |
| 记忆可以物质化 | 用回忆盖房子，住进去能看到房主的一生 |
| 文字写出来就有效力 | 写"墙"就会出现墙（但要消耗"墨水"这种稀有资源） |
| 金属在这个世界是液态 | 没有铁器，一切坚硬工具需要替代方案 |

**生物类**
| 规则 | 对基建的影响 |
|------|-------------|
| 所有植物会走路 | 农田里的庄稼会跑，需要"植物围栏" |
| 尸体不会腐烂反而变硬 | 天然建材，但涉及伦理问题 |
| 动物的情绪是可见的彩色气体 | 可以收集"恐惧之气"做燃料 |
| 微生物有智慧 | 细菌可以谈判，发酵需要外交 |

### B2. 基建要素替换矩阵

| 维度 | 传统 | 科幻 | 奇幻 | 克苏鲁 | 历史穿越 |
|------|------|------|------|--------|---------|
| **能源** | 煤/石油 | 核能/反物质 | 灵石/元气 | SAN值/信仰 | 人力/畜力改良 |
| **建材** | 砖石/木材 | 合金/纳米材料 | 仙木/龙骨 | 凝固的疯狂 | 秦砖/夯土改良 |
| **动力** | 人/畜 | 机器人/AI | 傀儡/灵兽 | 触手/寄生体 | 水车/畜力杠杆 |
| **通讯** | 信鸽/烽火 | 量子通讯 | 传音法术 | 梦境共享 | 旗语/驿站改良 |
| **交通** | 马车/船 | 飞船/磁悬浮 | 传送阵/飞剑 | 空间折叠 | 驰道/水运改良 |
| **货币** | 金银/铜钱 | 数字货币 | 灵石/功德 | 记忆碎片 | 粮食本位/布帛 |

### B3. 脑洞组合公式

**公式**：`[反常识规则] × [现实基建需求] = 脑洞基建点子`

快速生成法——随机挑一条规则和一个需求组合：
- "声音是实体" × 建房 = 用歌声筑墙，不同音调的砖块硬度不同
- "时间因海拔而异" × 农业 = 在山顶建加速温室，1天收一季粮食
- "植物会走" × 防御 = 训练树木列队当城墙，还能自动追击入侵者
- "影子是实体" × 能源 = 中午影子最短=能量最集中，收集影子做电池
- "颜色有温度" × 冶炼 = 把铁矿石涂红就能加热到熔点

### C3. 称号体系（由低到高）

随着迪化事件积累，NPC给主角的称号应该升级：

| 阶段 | 称号等级 | 命名模式 | 示例 |
|------|---------|---------|------|
| Level 1 | 技能型 | "[能力]+匠/师" | "无声建造师"、"控水匠" |
| Level 2 | 神话型 | "[领域]+之主/掌控者" | "万火之主"、"引力掌控者" |

| Level 4 | 终极型 | "[抽象概念]+神/创造者" | "秩序之神"、"文明缔造者" |

**叠加规则**：NPC不会忘记旧称号，而是叠加。到后期主角的全称会变得超长：
> "无声建造师、万火之主、净秽真君、引力解放者、天蓬大元帅（猪封的）"

## 模块 D：NPC 反应模板库

### D1. 基础反应五阶段

| 阶段 | 情绪 | 身体语言 | 典型台词模板 |
|------|------|---------|-------------|
| 1 震惊 | 大脑宕机 | 瞳孔放大，嘴巴张开合不上 | "这……这……"（说不出完整的话） |
| 2 恐惧 | 本能退缩 | 后退，手挡在面前，低头不敢看 | "妖……妖术！快跑！" |

### D2. 不同身份的反应差异

| NPC类型 | 关注点 | 第一反应 | 后续行为 |
|---------|-------|---------|---------|
| **老者/长老** | 这是否符合古老传说 | 翻箱倒柜找古籍验证 | 成为主角的"宣传部长" |
| **战士/将军** | 这能不能打仗用 | 拿起来就想往敌人头上砸 | 主动请求为主角打仗换技术 |
| **祭司/巫师** | 这是否冒犯神灵 | 警告主角会遭天谴 | 要么成为最大阻力，要么改信主角 |
| **商人/掮客** | 这能卖多少钱 | 眼睛放光开始算账 | 自告奋勇当"独家代理" |
| **小孩** | 好不好玩 | 毫无恐惧伸手就摸 | 成为主角最早的"学徒" |
| **统治者** | 这能否巩固我的权力 | 表面赞赏内心盘算 | 要么招揽要么除掉 |
| **敌人/对手** | 这东西怎么破 | 震惊后转为研究 | 偷技术/绑架主角/正面强攻 |
| **流民/难民** | 能不能活下去 | 哭着跪下求收留 | 最忠诚但也最脆弱的支持者 |

### D3. 群体反应的传播链

单个NPC的反应会扩散：

```
目击者（1人）→ 亲友圈讨论（5-10人）→ 村庄集体围观（50-100人）
   → 传说化（添油加醋传到隔壁村）→ 朝廷/高层势力听闻 → 派人来调查
```

**传播中的变形**：每传一层，事实会夸大一倍。
- 真实："主角修了个抽水马桶"
- 第1层："主角召唤了一个能吃掉一切脏东西的精灵"
- 第2层："主角在家里养了一只吞天巨兽，粪便都不够它吃的"
- 第3层："主角驯服了地下世界的远古凶兽，正在用它挖掘通往地心的隧道"

## 模块 E：经典基建文案例拆解

### E1. 爆款基建文的共性

分析市面上成功的基建文，提炼共同结构：

| 要素 | 成功作品的共性 | 失败作品的共性 |
|------|-------------|-------------|
| **开局** | 30章内完成第一个标志性建设 | 前50章还在铺设定 |
| **金手指** | 有明确限制，需要动脑用 | 万能系统，要啥有啥 |
| **NPC** | 有鲜明性格，不是纯工具人 | 所有NPC都是"跪族" |
| **节奏** | 建设-危机-建设-危机交替 | 纯建设流水账 |
| **爽点** | 成果展示时有仪式感 | 一笔带过"建好了" |
| **反派** | 反派也在发展，构成压力 | 反派纯送人头 |

### E2. 常见题材模板

**模板 1：历史穿越基建**
```
背景：现代人穿越到古代
金手指：现代知识 + 某种辅助系统
节奏：种地→修路→建工厂→打仗→建国
核心卖点：用现代科技降维打击古人的认知
典型读者预期：考据+爽+搞笑
```

**模板 2：异世界基建**
```
背景：穿越/转生到异世界（魔法/修仙/克苏鲁）
金手指：某种制造/转化能力
节奏：生存→聚集人口→建立据点→对抗原生势力
核心卖点：用工业思维解决魔法问题
典型读者预期：脑洞+世界观+成就感
```

**模板 3：末世重建**
```
背景：灾后/末世环境
金手指：废墟中找到的某种核心科技
节奏：生存→寻找幸存者→重建社区→恢复文明
核心卖点：废土美学+人性光辉
典型读者预期：紧张+感动+建设成就
```

**模板 4：游戏/系统基建**
```
背景：游戏世界具现化或真实世界出现游戏系统
金手指：领主系统/建造系统
节奏：跟着系统任务走，但要处理系统之外的变数
核心卖点：数据化的满足感（+1, +1, +1...）
典型读者预期：清晰的成长感+惊喜的任务奖励
```

### E3. 节奏对标参考

| 章节范围 | 应该完成的里程碑 | 情绪曲线 |
|---------|----------------|---------|
| 1-5 | 交代设定+主角陷入困境+金手指激活 | 好奇→绝望→希望 |
| 5-15 | 完成第一个建设成果+第一个迪化事件 | 紧张→爽→期待 |
| 15-30 | Level 1完成，第一次小危机 | 积累→爆发→新开始 |
| 30-60 | Level 2完成，人口增长，外部势力关注 | 日常→惊喜→压力 |
| 60-100 | Level 3推进，第一次大危机 | 膨胀→崩塌→重建 |
| 100-150 | 格局打开，从领主变国王 | 野心→挑战→成就 |
| 150+ | 终极危机+金手指真相+最终决战（基建式） | 绝望→全力一搏→史诗 |

## 模块 F：常见雷区与解决方案

### F1. 致命雷区（踩了必扑街）

| 雷区 | 症状 | 根因 | 解决方案 |
|------|------|------|---------|
| **无限资源** | 主角要什么有什么，毫无紧张感 | 金手指没有限制 | 回到 Phase 3，加限制/代价 |
| **NPC工具人化** | 所有NPC都是"好的主人""遵命主人" | 没有给NPC独立动机 | 给每个NPC至少1个与主角冲突的需求 |
| **科技跳级** | 石器时代直接造电脑 | 跳过了中间的产业链 | 用模块A3的科技树倒推缺什么前置 |
| **纯流水账** | 建了A，建了B，建了C，读者昏睡 | 没有危机打断建设节奏 | 每完成1个大建设必须触发1个新问题 |
| **主角全能** | 主角什么都会修什么都会造 | 没有设计主角的知识盲区 | 给主角一个致命弱点（比如不懂化学/不会打仗） |

### F2. 常见瑕疵（影响体验）

| 瑕疵 | 症状 | 解决方案 |
|------|------|---------|
| **节奏太慢** | 一个厕所写了3章 | 压缩技术细节，保留爽点场景 |
| **节奏太快** | 3章建了一座城 | 放慢，补上材料采集和施工过程 |
| **迪化过度** | 每章都在装逼，审美疲劳 | 控制迪化密度（见Phase 5.3） |
| **迪化不足** | 建了很多东西但没人在乎 | 每个大建设必须配一个NPC反应场景 |
| **地图模糊** | 读者不知道领地长什么样 | 每10章附一次"领地现状描述"（不用画图，文字描述即可） |
| **数据缺失** | 读者不知道有多少人/多少粮食 | 关键数据随文交代（人口、粮食储备、建材库存） |

### F3. 自查清单

每写完一个建设弧（约5-10章），对照检查：

- [ ] 这个建设项目有完整的产业链支撑吗？
- [ ] 建设过程中遇到了至少1个意外困难吗？
- [ ] 有至少1个NPC对成果做出了有个性的反应吗？
- [ ] 建设成果的展示用了感官描写（视/听/触/嗅/味至少3种）吗？
- [ ] 这次建设为后续剧情埋了至少1个伏笔/副作用吗？
- [ ] 主角在这个过程中有成长/变化吗？（不只是技术，也包括心态）
- [ ] 读者在这一段能清楚知道领地现在的整体样貌吗？
